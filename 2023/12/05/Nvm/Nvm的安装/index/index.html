# Nvm 的作用

我们可能在工作和学习中都会遇到需要使用多个 node 版本的时候，nvm 正为解决这个而生，NVM（Node Version Manager）是一个用于管理 Node.js 版本的工具，并没有官方网站。NVM 是一个开源项目，其代码托管在 GitHub 上

# Nvm 下载与安装

注意：
1. 首先安装目录不要有空格和中文，会出现乱码！
2. 如果已经有 node，需要卸载当前 node，不然会产生报错或异常情况！卸载直接通过控制面板卸载即可！

Nvm 的 github 地址：

https://github.com/coreybutler/nvm-windows/releases

安装：
找到 nvm-setup.zip 或者 nvm-setup.exe 除了路径正常安装即可

选择安装路径：
默认是 C 盘，我自己指定的是 D 盘的 nvm 文件夹，Nvm 和以后所有 Node 版本会下载到你指定的文件夹
（我是为了防止成为 C 盘战士）

设定当前系统使用 node 版本的存放位置：
也就是说系统最终使用的是这个文件夹里面的 node 版本，nvm 下载了多个版本，指定某个版本后，这个指定的版本就会放到这个 node 文件中，供系统使用。（在这里我也放到了 D 盘的 Node 文件夹下）

Npm 配置淘宝镜像源：
第一种方式：
1. 设置淘宝镜像源

npm config set registry https://registry.npmmirror.com

2. 查看镜像使用状态

npm config get registry

3. 如果返回 https://registry.npmmirror.com，说明配置的是淘宝镜像。

第二种方式：
1. 安装 nrm 镜像管理工具

npm i nrm -g

2. 使用 nrm 切换镜像

nrm use xxx

3. 查看是否切换成功（镜像后边带 * 表明正在使用这个镜像）

nrm ls

查询可以下载的所有 node 版本：
首先，使用 NVM 命令 nvm list available，查看可安装的 NodeJS 版本。建议选择 LTS 中的版本进行安装
（LTS，即 Long Term Support，长期支持版本）

nvm list available

安装指定版本：

nvm install xxx

查看已经安装的 node 版本：

nvm list

切换 Node 版本：
注意：切换版本前请以管理员身份运行命令提示符，否则会出现以下错误↓
